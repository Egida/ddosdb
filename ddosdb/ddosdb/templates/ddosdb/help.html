Help and information

Query language

DDoSDB uses the MongoDB Query Language (MQL). protocol: "DNS" will match all DNS attacks, (protocol: "DNS" OR protocol: "UDP") AND NOT (src_ips: "127.0.0.1") matches all DNS and UDP attacks that are not from IP 127.0.0.1. dst_ports: [10 TO 50] matches all attacks where the destination port is in the range 10 to 50.

Sorting

DDoSDB follows Elasticsearch sorting conventions. The default sorting method is _score, which uses the internal score of the match. More complex queries can be constructed, like file_type:asc,duration_sec:desc, which will sort by file type in ascending order first, then sort for the duration in descending order. Certain fields may require you to search by the content instead of the key, like start_time.keyword:desc
